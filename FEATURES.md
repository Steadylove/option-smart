# 期权卖方策略 AI Agent — 功能清单

## 项目概述

面向个人期权卖方交易者的 AI 辅助分析工具，接入长桥证券期权行情，提供希腊值分析、仓位建议、止盈止损/展期提醒和 AI 策略顾问。本地部署于 Mac，配合 Telegram 推送告警。

### 核心定位

- **策略方向**：期权卖方（收租），赚取时间价值衰减
- **标的范围**：TQQQ、TSLL、NVDL 等高波动、高流动性杠杆 ETF
- **覆盖策略**：Cash-Secured Put / Covered Call / Credit Spread / Iron Condor / Strangle
- **AI 模型**：GLM-5（智谱 AI）
- **部署方式**：本地 Mac
- **通知渠道**：Web 页面 + Telegram Bot

## 模块一：数据接入层

### 1.1 长桥 API 对接

- [ ] 接入长桥 OpenAPI SDK（Python）
- [ ] 获取标的实时行情（价格、涨跌幅、成交量）
- [ ] 获取期权链数据（按到期日 + 行权价）
- [ ] 获取期权实时报价（买卖价、IV、未平仓量、成交量）
- [ ] WebSocket 订阅实时行情推送
- [ ] 获取账户持仓数据（期权 + 正股）

### 1.2 希腊值计算引擎

长桥客户端 UI 上会显示希腊值，但 OpenAPI 接口不返回 Delta/Gamma/Theta/Vega。
API 已返回计算所需的全部原料（标的价格、行权价、到期日、IV），使用 `mibian` 或 `py_vollib` 库基于 Black-Scholes 模型自行计算即可，开发成本很低：

- [ ] Delta — 标的价格变动 $1 时期权价格的变动量
- [ ] Gamma — Delta 本身的变化速度
- [ ] Theta — 每天时间价值的衰减量（卖方最关心）
- [ ] Vega — IV 变动 1% 时期权价格的变动量
- [ ] Rho — 利率变动的影响（优先级低）
- [ ] 组合级别希腊值汇总（所有持仓的加总 Delta/Gamma/Theta/Vega）

### 1.3 IV 分析

- [ ] 计算 IV Rank（当前 IV 在过去 52 周中的百分位排名）
- [ ] 计算 IV Percentile（过去 52 周中低于当前 IV 的天数占比）
- [ ] IV 历史走势图
- [ ] 高 IV 标的筛选提醒（IV Rank > 50% 时适合卖权）

## 模块三：期权链分析器

### 3.1 期权链展示

- [ ] 按到期日分组展示完整期权链
- [ ] 每个合约显示：Delta、Gamma、Theta、Vega、IV、OI、成交量、Bid/Ask
- [ ] 虚值/平值/实值颜色区分
- [ ] 支持筛选（按 DTE 范围、Delta 范围、最低成交量）

### 3.2 合约推荐引擎

根据不同策略，自动推荐最优合约：

**Cash-Secured Put（CSP）：**
- [ ] 推荐 Delta -0.2 ~ -0.3 的 OTM Put（约 70-80% 胜率）
- [ ] 计算年化收益率（权利金 / 保证金 × 365 / DTE）
- [ ] 标注"甜蜜点"：DTE 30-45 天（Theta 衰减最优区间）

**Covered Call：**
- [ ] 推荐 Delta 0.2 ~ 0.3 的 OTM Call
- [ ] 计算被行权时的总收益（股价涨幅 + 权利金）
- [ ] 标注是否持有足够正股

**Credit Spread（牛市 Put Spread / 熊市 Call Spread）：**
- [ ] 推荐宽度（$1 / $2 / $5 价差）
- [ ] 计算最大收益、最大亏损、盈亏比
- [ ] 计算胜率（基于 Delta）

**Iron Condor：**
- [ ] 同时推荐 Put Spread + Call Spread
- [ ] 计算盈利区间（两个行权价之间）
- [ ] 适合 IV Rank 高 + 预期震荡行情

**Strangle：**
- [ ] 推荐两侧行权价
- [ ] 计算盈亏平衡点
- [ ] 保证金需求计算

### 3.3 对比分析

- [ ] 多合约横向对比（收益率、风险、胜率）
- [ ] 不同 DTE 的 Theta 衰减曲线对比
- [ ] 盈亏概率分布图（基于 IV 的蒙特卡洛模拟）

- [ ] 分散化建议（避免同方向集中暴露）

## 模块五：持仓诊断与智能操作建议（Position Doctor）

这是整个系统最核心的模块。对每个持仓做深度"体检"，并给出专业级的操作建议，不是简单的"赚了平、亏了跑"，而是像专业期权交易员一样分析每个维度后再做决策。

### 5.1 单持仓健康诊断

每个期权持仓生成一份"健康报告卡"：

**基础状态指标：**
- [ ] 持仓健康度评分（绿 / 黄 / 红，综合多维度打分）
- [ ] 当前状态分类：安全区（OTM 远离行权价）/ 警戒区（接近 ATM）/ 危险区（ITM）
- [ ] 盈亏百分比 + 绝对值
- [ ] 距到期天数（DTE）及 Theta 衰减阶段标注（慢速期 / 加速期 / 极速期）
- [ ] 当前 Delta 对应的被行权概率

**盈亏归因分解（P&L Attribution）：**
把当前盈亏拆解为四个驱动因素，搞清楚你到底是怎么赚/亏的：
- [ ] Theta 贡献 — 时间衰减帮你赚了多少（卖方的核心利润来源）
- [ ] Delta 贡献 — 标的价格变动导致的盈亏
- [ ] Vega 贡献 — IV 变化导致的盈亏（IV 下降对卖方有利）
- [ ] Gamma 贡献 — 非线性风险带来的额外盈亏
- [ ] 可视化：堆叠柱状图，一眼看出谁在帮你赚钱、谁在拖后腿

**时间价值分析：**
- [ ] 剩余时间价值 vs 已衰减时间价值（进度条）
- [ ] 剩余时间价值衰减速率预测（未来 7 天每天预计赚多少 Theta）
- [ ] 时间价值 vs 持有风险的性价比评估（剩余收益是否值得冒剩余风险）

**盈利概率追踪：**
- [ ] 实时盈利概率（Probability of Profit），基于当前 IV 和 DTE 计算
- [ ] 与开仓时的盈利概率对比（概率是变好了还是变差了）
- [ ] 最大盈利概率路径预测（如果标的横盘/缓涨/缓跌的不同情景）

### 5.2 组合维度分析（Portfolio-Level）

不只看单个持仓，从整个组合角度评估风险暴露：

**希腊值仪表盘：**
- [ ] 组合总 Delta（方向性风险）— 用仪表盘表示偏多/偏空/中性
- [ ] 组合总 Gamma（凸性风险）— 大波动时亏损加速的程度
- [ ] 组合总 Theta（每日时间收入）— 你的"日租金"是多少
- [ ] 组合总 Vega（波动率风险）— IV 涨 1% 你亏多少
- [ ] Beta-Weighted Delta（以 SPY 为基准归一化）— 统一度量不同标的的方向风险

**集中度与相关性：**
- [ ] 按标的的仓位集中度饼图（TQQQ / TSLL / NVDL 各占多少）
- [ ] 标的间相关性矩阵热力图（这三个票往往同涨同跌，风险高度集中）
- [ ] 按方向的集中度（卖 Put 占比 vs 卖 Call 占比 vs Spread 占比）
- [ ] 按到期日的集中度（是否大量合约集中在同一周到期）
- [ ] 集中度风险告警（如 80% 仓位都是卖 Put → 单边暴跌将全线亏损）

**资金效率：**
- [ ] 保证金占用率（已用 / 总可用）
- [ ] 每 $1000 保证金的日 Theta 收入（资金效率指标）
- [ ] 各持仓的资金效率排名（哪个持仓性价比最高/最低）
- [ ] 闲置资金提示（如果有大量未使用保证金，提示可以开新仓）

### 5.3 情景压力测试（Scenario Analysis）

模拟各种市场极端情况，提前知道"最坏会怎样"：

**标的价格冲击：**
- [ ] 标准情景：标的 ±3% / ±5% / ±10% / ±20% 时的组合盈亏
- [ ] 极端情景：2020.03 级别暴跌（-30%）、熔断情景
- [ ] 按标的分别模拟（TQQQ 跌 10% 但 TSLL 涨 5% 的混合情景）
- [ ] 结果展示：盈亏金额 + 保证金变化 + 是否触发 Margin Call

**IV 冲击：**
- [ ] IV 暴涨情景：+30% / +50% / +100%（恐慌事件，VIX 飙升）
- [ ] IV 暴跌情景：-20% / -30%（IV Crush，财报后）
- [ ] 对卖方的影响量化（IV 涨 = 卖方浮亏，即使标的没动）

**时间推移模拟：**
- [ ] T+7 / T+14 / T+30 天后的持仓状态预测（假设标的不动）
- [ ] 各合约的 Theta 衰减曲线预览
- [ ] "躺赚"测算：如果什么都不做，到期时预计赚多少

**复合压力测试：**
- [ ] 最坏情景：标的暴跌 + IV 暴涨（卖 Put 的噩梦）
- [ ] 次坏情景：标的暴涨 + IV 暴涨（卖 Call 的噩梦）
- [ ] 黑天鹅预案：如果出现极端情景，需要多少资金来应对
- [ ] 交互式"What-If"工具：手动输入价格 / IV 变化，实时计算影响

### 5.4 操作决策矩阵（Action Decision Matrix）

对每个持仓，同时计算所有可能的操作方案并横向对比，而不是只给一个建议：

| 操作方案 | 计算项 |
|---------|--------|
| **继续持有到期** | 期望收益、盈利概率、最大风险、剩余 Theta 收入 |
| **现在平仓** | 锁定盈亏金额、释放保证金额度、资金可再投收益 |
| **向后展期（Roll Out）** | 平仓成本 + 新合约权利金 = 净信用/借记、新到期日、新盈亏平衡点 |
| **向下/上展期（Roll Out & Down/Up）** | 净信用/借记、新行权价、新 Delta、新盈利概率 |
| **加保护腿（转为 Spread）** | 买入保护期权的成本、最大亏损封顶值、新盈亏图 |
| **加仓（Double Down）** | 新均价、新保证金需求、风险是否可控 |
| **对冲（买标的/买反向期权）** | Delta 中性化成本、对冲后的新组合希腊值 |

每个方案给出：
- [ ] 预期收益 / 最大亏损
- [ ] 盈利概率变化（操作前 vs 操作后）
- [ ] 资金效率变化
- [ ] 风险调整后收益（Sharpe-like 评分）
- [ ] 直观评分：⭐⭐⭐⭐⭐（推荐度）

### 5.5 AI 智能操作推荐

AI 综合以上所有分析，生成排序后的操作建议清单：

**推荐卡片格式（每个持仓一张）：**
```
┌─────────────────────────────────────────────┐
│ 🔴 TQQQ 03/21 $60 Put (Sell)    紧急度: 高  │
├─────────────────────────────────────────────┤
│ 当前状态: 亏损 -35%，Delta -0.48，DTE 12    │
│ 健康度: 危险区（接近 ATM）                    │
│                                             │
│ 推荐操作: 向后+向下展期                       │
│ 具体方案: 平 03/21 $60P → 卖 04/17 $55P     │
│ 净信用: +$0.45/张                            │
│ 新盈利概率: 72% (当前: 52%)                   │
│ 新盈亏平衡点: $54.55 (当前: $58.80)           │
│                                             │
│ 推荐理由:                                    │
│ 1. DTE 仅剩 12 天，Gamma 风险急剧上升        │
│ 2. 展期可收到净信用，降低成本基础              │
│ 3. 行权价下移 $5，盈利概率从 52% → 72%       │
│ 4. 多出 27 天时间，Theta 继续为你工作         │
│                                             │
│ 备选方案:                                    │
│ • 直接止损 → 亏损 $180，释放 $6000 保证金    │
│ • 加保护腿 $55P → 最大亏损封顶 $500          │
│ • 继续持有 → 盈利概率 52%，风险敞口 $6000    │
└─────────────────────────────────────────────┘
```

**推荐排序逻辑：**
- [ ] 紧急度排序：先处理红色（危险）持仓，再处理黄色（警戒）
- [ ] 时间敏感排序：DTE 最短的优先
- [ ] 风险排序：Delta 绝对值最大的优先（最容易被行权的）
- [ ] 盈利锁定排序：已达止盈目标但还没平的合约优先提醒

**批量操作建议：**
- [ ] 组合级别再平衡建议（如"整体偏多，建议卖一组 Call Spread 对冲"）
- [ ] 到期日集中问题的分散建议（如"5 个合约都在 03/21 到期，建议分散"）
- [ ] 资金优化建议（如"平掉这 2 个低效持仓，腾出资金开更优的新仓"）

### 5.6 持仓生命周期追踪

从开仓到平仓，完整追踪每个持仓的生命周期：

- [ ] 开仓时记录：开仓理由、当时的 IV Rank、标的价格、希腊值快照
- [ ] 每日快照：自动记录每日收盘时的持仓状态（价格、希腊值、盈亏）
- [ ] 关键事件标注：触发止盈线、进入警戒区、展期操作等
- [ ] 平仓时汇总：持仓天数、最终盈亏、年化收益率、策略复盘
- [ ] 生命周期可视化时间线（开仓 → 持有 → 调仓 → 平仓的完整路径）

## 模块六：止盈 / 止损 / 展期提醒

### 6.1 止盈提醒

- [ ] 权利金盈利达 50% 时推送第一档提醒
- [ ] 权利金盈利达 75% 时推送第二档提醒
- [ ] 计算持有到期 vs 现在平仓的期望收益对比
- [ ] 建议：平仓锁定利润 → 释放资金 → 开新仓（资金周转效率）

### 6.2 止损提醒

- [ ] 亏损达权利金的 2x 时告警（如收 $1 权利金，亏损达 $2）
- [ ] 标的价格突破行权价时紧急告警
- [ ] 评估止损 vs 展期的利弊：
  - 止损：直接平仓认亏，释放保证金
  - 展期（Roll）：平当前合约 + 开下月更远行权价的合约
- [ ] AI 给出建议（基于当前 IV、DTE、标的走势判断）

### 6.3 展期（Roll）建议

- [ ] 当合约进入 ITM（实值）且 DTE < 14 时，提示是否展期
- [ ] 计算展期信用/借记（Roll for credit 还是 debit）
- [ ] 推荐展期目标合约（下月、更远行权价）
- [ ] 展期后新的盈亏平衡点计算
- [ ] 展期 vs 认赔 vs 接货（被行权）的三方对比分析

### 6.4 被行权处理建议

- [ ] Sell Put 被行权 → 建议是否转为 Covered Call（车轮策略 Wheel）
- [ ] Sell Call 被行权 → 提示股票被 Call 走后的下一步
- [ ] 计算实际持仓成本（行权价 - 收到的权利金）

## 模块八：期权知识库

### 8.1 基础概念

- [ ] 期权是什么？Call vs Put

### 8.2 希腊值详解

- [ ] **Delta 详解**
  - 定义：标的价格变动 $1，期权价格变多少
  - 卖方视角：Delta 越小（绝对值），越安全
  - 作为胜率的近似：Delta 0.3 ≈ 30% 被行权概率 ≈ 70% 胜率
  - 组合 Delta：衡量你的方向性暴露
- [ ] **Gamma 详解**
  - 定义：Delta 的变化速度
  - 卖方视角：Gamma 是敌人，价格剧烈波动时亏损加速
  - ATM 期权 Gamma 最大，OTM 和 ITM 较小
  - 临近到期 Gamma 会急剧增大（Gamma Risk）
- [ ] **Theta 详解**
  - 定义：每天时间价值衰减多少
  - 卖方视角：Theta 是朋友，每天都在帮你赚钱
  - 最后 30 天 Theta 加速衰减
  - ATM 期权 Theta 最大
  - DTE 30-45 天是卖权的甜蜜点
- [ ] **Vega 详解**
  - 定义：IV 变动 1% 时期权价格变多少
  - 卖方视角：希望 IV 下降（Vega 为负收益）
  - 高 IV 时卖权 → IV 回落时额外赚钱（IV Crush）
  - 财报前后的 IV Crush 现象
- [ ] **Rho 详解**
  - 定义：利率变动对期权价格的影响
  - 实际交易中影响最小，了解即可

### 8.3 卖方策略教学

每个策略包含：图解 + 盈亏图 + 适用场景 + 风险提示 + 实例

- [ ] **Cash-Secured Put（CSP）**
  - 原理：卖出 Put，准备好现金接货
  - 适用场景：看好标的，愿意低价买入
  - 收益上限：收到的权利金
  - 最大亏损：行权价 × 100 - 权利金
  - 选择行权价的经验法则
  - 选择到期日的经验法则
- [ ] **Covered Call（备兑开仓）**
  - 原理：持有 100 股 + 卖出 Call
  - 适用场景：持股且短期不看大涨
  - 收益上限：(行权价 - 持仓成本) × 100 + 权利金
  - 被 Call 走的处理方式
- [ ] **Wheel 策略（车轮策略）**
  - CSP → 被行权接货 → Covered Call → 被行权交股 → 循环
  - 完整流程图解
  - 适合标的的特征
- [ ] **Bull Put Spread（牛市 Put 价差）**
  - 卖近买远，限制最大亏损
  - 保证金更低，适合小资金
  - 盈亏计算方法
- [ ] **Bear Call Spread（熊市 Call 价差）**
  - 卖近买远 Call 端
  - 适用于看跌或不看涨行情
- [ ] **Iron Condor（铁秃鹰）**
  - 同时卖 Put Spread + Call Spread
  - 赚取区间震荡，两侧收租
  - 如何选择宽度和行权价
  - 调仓方法（被测试时）
- [ ] **Strangle（宽跨式）**
  - 同时卖裸 Put + 裸 Call
  - 收入最高但风险最大
  - 适合高 IV 环境

## 模块九：市场事件日历（Event Calendar）

对期权卖方来说，重大事件 = 波动率飙升 = 风险和机会并存。这个模块帮你提前感知风险、合理避坑或主动利用。

### 9.1 事件数据源

- [ ] 财报日历（Earnings Calendar）— 标的及其底层成分股的财报发布日期
- [ ] 美联储议息会议（FOMC）日期及市场预期
- [ ] CPI / PPI / 非农等关键经济数据发布日
- [ ] 期权到期日（月度 / 周度 OpEx）
- [ ] 重大政策事件（关税、制裁、监管等，通过新闻 API 抓取）
- [ ] 股票拆分 / 分红除权日
- [ ] 数据来源：新闻 API（如 NewsAPI / Finnhub）+ 财经日历 API + AI 摘要

### 9.2 事件看板

- [ ] 日历视图：按周/月展示即将到来的事件
- [ ] 事件与标的关联标注（如"NVDA 财报" 直接影响 NVDL）
- [ ] 事件距今倒计时
- [ ] 事件对 IV 的历史影响（如过去 4 次财报前 IV 分别涨了多少）
- [ ] 事件影响等级标签（高/中/低）

### 9.3 事件驱动的策略提醒

- [ ] 财报前提醒：持仓标的即将发财报，建议提前平仓或对冲
- [ ] FOMC 前提醒：利率决议前波动率通常上升，提示风险
- [ ] 财报后 IV Crush 机会：财报公布后 IV 暴跌，适合卖权的窗口
- [ ] 大事件前不建议新开裸卖仓位的告警
- [ ] Telegram 推送事件提醒（提前 1-3 天）

### 9.4 新闻聚合

- [ ] 标的相关新闻实时聚合（TQQQ / TSLL / NVDL 及底层标的）
- [ ] AI 摘要：将多条新闻浓缩为一句话要点
- [ ] 新闻情绪标签（利多 / 利空 / 中性）
- [ ] 突发重大新闻 Telegram 告警

## 模块十：复盘分析（Post-Mortem）

大涨大跌之后，搞清楚"为什么"比"赚了亏了"更重要。这个模块帮你积累经验、优化策略。

### 10.1 异动检测

- [ ] 自动检测标的日内涨跌幅超阈值（如 ±5%）
- [ ] 自动检测 IV 单日变化超阈值（如 ±10%）
- [ ] 检测到异动后自动触发复盘流程
- [ ] Telegram 推送异动告警 + 复盘摘要

### 10.2 AI 归因分析

异动发生后，AI 自动分析可能原因：

- [ ] 关联当天新闻事件（财报、政策、宏观数据等）
- [ ] 关联底层标的走势（如 NVDL 暴跌是否因为 NVDA 业绩不及预期）
- [ ] 关联大盘走势（是个股原因还是系统性风险）
- [ ] 分析 IV 变化是否合理（恐慌性抛售 vs 理性调整）
- [ ] 生成结构化复盘报告：
  - 事件概述
  - 涨跌幅 / IV 变化 / 成交量异常
  - 可能原因（按置信度排序）
  - 对你持仓的实际影响（盈亏变化、希腊值变化）
  - 后续建议（加仓/减仓/展期/观望）

### 10.3 交易日志与复盘记录

- [ ] 自动记录每笔交易（开仓/平仓/展期/被行权）
- [ ] 每笔交易附带当时的市场快照（标的价格、IV、希腊值）
- [ ] 手动添加复盘笔记（你当时的判断和情绪）
- [ ] 历史交易胜率 / 平均盈亏 / 年化收益统计

### 10.4 策略回顾

- [ ] 按策略类型统计表现（CSP / CC / Spread 哪个赚得最多）
- [ ] 按标的统计表现（TQQQ / TSLL / NVDL 哪个收租效率最高）
- [ ] 按 DTE 统计表现（30 天 vs 45 天哪个更优）
- [ ] 按 IV Rank 开仓时机统计（高 IV 开仓是否确实更赚）
- [ ] AI 周度/月度总结报告：复盘这段时间的操作，给出改进建议
